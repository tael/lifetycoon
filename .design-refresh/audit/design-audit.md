# 🎨 인생타이쿤 비주얼 디자인 리프레시 진단 보고서 (Design Audit)

**작성자:** 시니어 프로덕트 디자이너
**대상:** 초등 고학년 경제 교육 게임 '인생타이쿤'
**목적:** 투박한 UI 개선 및 사용자(어린이) 몰입도 향상을 위한 디자인 전략 수립

---

## 1. "투박함"의 원인 진단 (8가지 핵심 항목)

현재 디자인이 "투박하다"는 피드백을 받는 구체적인 기술적/미학적 이유는 다음과 같습니다.

1.  **유니코드 이모지 과의존:** 모든 아이콘과 시각 요소가 시스템 이모지로 구성되어 있어, OS별로 모습이 다르고 게임만의 독창적인 그래픽 에셋이 부족해 보입니다. (가장 큰 원인)
2.  **타이포그래피 위계(Hierarchy) 부족:** 모든 텍스트에 배민 폰트(BMJUA/BMDOHYEON)를 적용하여 제목과 본문의 구분이 모호하고, 특히 숫자가 많은 경제 게임임에도 가독성이 낮은 폰트로 숫자를 표기하고 있습니다.
3.  **인라인 스타일의 남용:** `tokens.css`가 정의되어 있음에도 실제 컴포넌트(`PlayScreen.tsx` 등) 내부에 `gap: 6`, `padding: '6px 0'` 등 하드코딩된 스타일이 많아 시각적 리듬이 불규칙합니다.
4.  **평면적인 카드 디자인:** 단순한 흰색 배경에 아주 옅은 그림자(`--shadow-sm`)만 사용되어, '게임'다운 입체감이나 손에 잡히는 듯한 '장난감 같은(Tactile)' 느낌이 부족합니다.
5.  **정보 밀도 조절 실패:** `PlayScreen`의 홈 탭 등에 너무 많은 정보와 버튼이 한꺼번에 나열되어 있어, 어린 사용자가 어디에 먼저 주목해야 할지 알기 어렵습니다.
6.  **버튼의 인터랙션 부재:** 버튼이 단순히 둥근 사각형 형태이며, 클릭 시 물리적인 눌림 효과나 그림자 변화가 적어 '누르는 맛'이 떨어집니다.
7.  **배경과 콘텐츠의 충돌:** 나이에 따라 변하는 배경 그라데이션은 좋으나, 그 위의 카드들과 대비가 충분하지 않거나 색 조합이 복잡해 보일 때가 있습니다.
8.  **아이콘 크기 체계 부재:** `1.2rem`, `2.2rem`, `5rem` 등 이모지 크기가 명확한 규칙 없이 배치되어 화면의 정렬이 깨져 보입니다.

## 2. 전반적 개선 방향 (General Design Direction)

| 항목 | 개선 방향 |
| :--- | :--- |
| **컬러 (Color)** | 파스텔톤 배경은 유지하되, 주요 상호작용 요소에는 **비비드한 강조색**을 사용하여 '포인트'를 명확히 함. |
| **타이포 (Typo)** | **제목은 배민 폰트**, **본문 및 숫자/데이터는 프리텐다드(Pretendard)**를 사용하여 정보 가독성을 획기적으로 개선. |
| **스페이싱 (Spacing)** | 인라인 스타일을 제거하고 `tokens.css`의 8px 단위 그리드 시스템으로 정렬을 일원화. |
| **카드/모달 (Card)** | 테두리 두께를 키우고(2px+), 그림자를 더 짙고 선명하게(Hard Shadow) 사용하여 **팝업북 같은 느낌** 부여. |
| **버튼 (Button)** | 하단에 3~4px의 어두운 테두리를 추가한 **3D 스타일 버튼** 도입 (클릭 시 밑으로 내려가는 효과). |
| **탭바 (TabBar)** | 단순한 하단 바가 아닌, **게임 HUD(Heads-up Display)** 느낌의 플로팅 스타일이나 아이콘 중심 디자인 제안. |
| **차트 (Chart)** | 선 두께를 키우고 점(Dot)을 더 크게 디자인하여 데이터가 '놀이'처럼 보이게 개선. |
| **애니메이션** | 단순한 Fade-in 외에 Bounce(통통 튀는) 효과를 적극 사용하여 생동감 부여. |
| **일러스트/아이콘** | 이모지를 보조 수단으로 내리고, **SVG 벡터 아이콘(Lucide-react 등)**을 메인 인터페이스에 도입. |

## 3. 우선순위 Top 10 실행 가능한 개선안

| 우선순위 | 항목 | 내용 | 관련 파일 |
| :--- | :--- | :--- | :--- |
| **1** | **폰트 이원화** | 제목(Display)과 본문(Body) 폰트 분리 적용. 숫자는 특히 가독성 높은 폰트로 변경. | `tokens.css` |
| **2** | **3D 버튼 시스템** | 모든 `.btn`에 하단 뎁스(Shadow-bottom) 추가 및 active 시 transform 효과 강화. | `mobile-first.css` |
| **3** | **아이콘 팩 도입** | Lucide-react 등 일관된 스타일의 SVG 아이콘 세트 도입 (이모지 대체). | `PlayScreen.tsx`, `TabBar` |
| **4** | **카드 테두리/그림자 강화** | `--radius-lg`를 기본으로 하고, 2px 외곽선과 짙은 그림자 적용. | `mobile-first.css` |
| **5** | **TabBar 디자인 리뉴얼** | 하단 고정 바를 투명도 있는 Blur 배경의 플로팅 형태로 변경. | `PlayScreen.tsx` |
| **6** | **상태 바(Stat) 시각화 개선** | 단순 progress bar를 넘어, 게이지 디자인에 아이콘과 그라데이션 적극 활용. | `PlayScreen.tsx` (StatMini) |
| **7** | **인라인 스타일 정리** | 하드코딩된 `gap`, `padding` 등을 CSS 클래스로 대체. | `TitleScreen.tsx`, `PlayScreen.tsx` |
| **8** | **숫자 포맷팅 강조** | 자산 금액 표시 시 단위(원)와 숫자의 크기/두께 차이를 주어 금액이 눈에 띄게 함. | `AssetCompositionBar.tsx` 등 |
| **9** | **모달 팝업 애니메이션** | `scale(0.8)`에서 `scale(1.05)`를 거쳐 `scale(1)`로 오는 **Spring 효과** 적용. | `mobile-first.css` |
| **10** | **배경 그라데이션 정돈** | 나이대별 배경색의 채도를 살짝 낮추고 콘텐츠 영역의 흰색 카드와 대비를 높임. | `PlayScreen.tsx` (ageGradient) |

---

## 결론 및 제언

현재의 디자인은 기능적으로는 훌륭하지만, 시각적으로는 '경제 교육 도구'에 더 가깝습니다. 사용자가 초등학생임을 고려할 때, 더 **'장난감 같고(Toy-like)'**, **'누르고 싶은(Tactile)'** 요소를 추가하는 것이 핵심입니다. 

위 10가지 개선안 중 **1(폰트), 2(버튼), 3(아이콘)**만 우선적으로 수정해도 "투박함"의 70% 이상이 사라질 것으로 판단됩니다.
